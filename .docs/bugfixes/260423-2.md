The obsolete "GetStudy connection refused propagates" case in D5 was removed.

Commit `da35f91`.
	- Required upstream/code changes to support non-study identifiers end-to-end:
		- `saga` package: `MLWHClient.AllSamples` (and by extension `AllSamplesForStudy`) is unusable against the production MLWH backend because `/integrations/mlwh/samples` with no filter returns HTTP 500. This either needs a fix in the SAGA service itself, or the client needs a scoped alternative — e.g. require a study/filter parameter, page smaller, or a new `FindSample(ctx, identifier)` endpoint that SAGA can resolve server-side instead of us downloading the whole table.
