# Bugfix 260501-8: Sample seqmeta hierarchical related data rendering

## Problem
The sample seqmeta details dialog for the nf-core/sarek sample fixture showed incorrect data:
- Direct metadata only showed `sampleid` when it should include `name`, `id_sample_lims`, `accession_number`, etc.
- Related data had 3 duplicate rows for the same study (name, identifier, accession) when it should show 1 study row
- Related data incorrectly showed 3 rows for different samples (sample-to-sample relations that shouldn't exist)
- Missing lanes section showing the sequencing lanes for the sample
- Missing hierarchical structure: should show 1 library row (parent) and 1 study row (grandparent of the sample)

## Root Cause
The frontend `buildDetailFields` and `buildHierarchicalGroups` functions in `seqmeta-badge.tsx` didn't handle sample metadata specially:
- `buildDetailFields` included sample-to-sample "Linked samples" for all metadata types, not just library metadata
- `buildHierarchicalGroups` only handled study and library metadata but not sample metadata
- The backend already provided `sample_detail` with lanes, sanger_id, sample_name, and the full sample object

## Solution
1. **Updated `buildDetailFields`**: Excluded "Linked samples" for sample metadata (only show them for library metadata) since samples don't directly relate to other samples.

2. **Updated `buildHierarchicalGroups`**: Added a new case for sample metadata that builds hierarchical groups for:
   - Parent library section (one row showing the library type)
   - Grandparent study section (one row showing the study the library belongs to)
   - Lanes section (multiple rows showing all sequencing lanes for the sample)

3. **Updated rendering logic**: When hierarchical groups exist, show both:
   - Direct metadata fields in a "Direct Metadata" section
   - Hierarchical groups in their respective sections (Library, Study, Lanes)

4. **Updated TypeScript types**: Extended `HierarchicalGroup` type to include `"library"` and `"lanes"` variants and their corresponding item types.

5. **Added rendering for new group types**: Implemented UI rendering for:
   - "library" groups: Show library type with copy/filter buttons
   - "lanes" groups: Show lane IDs (format: `{id_run}_{lane}#{tag_index}`) with copy buttons

## Tests Added
Added 3 regression tests in `frontend/tests/seqmeta-badge.test.ts`:

1. **"shows all direct metadata fields for a sample, not just sampleid"**: Verifies that sample direct metadata includes `sample_name`, `sanger_id`, `id_sample_lims`, and `accession_number`.

2. **"shows hierarchical related data for sample with library parent, study grandparent, and lanes"**: Verifies the hierarchical structure shows exactly:
   - One Library section with the library type
   - One Study section with the study name
   - One Lanes section with all sequencing lanes listed

3. **"does not show linked samples for a sample (no sample-to-sample relations)"**: Verifies that sample-to-sample "Linked samples" are not shown for sample metadata, even when the enrichment includes multiple samples in the `samples` array.

## Files Changed
- `frontend/components/seqmeta-badge.tsx`: Updated `buildDetailFields`, `buildHierarchicalGroups`, types, and rendering logic
- `frontend/tests/seqmeta-badge.test.ts`: Added 3 regression tests
