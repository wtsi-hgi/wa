# Bugfixes 2026-06-27 (4)

Export MLWH primitives that commit d285a154 built but left unexported, so the
planned MCP server (importing `github.com/wtsi-hgi/wa/mlwh`) can use
typed/compile-time access instead of fragile stringly-typed workarounds.
Additive only: must NOT change the served API, the Registry, or the generated
docs (api-reference.md / OpenAPI no-drift golden tests stay green unchanged).

- [x] BUG 1 — MLWH API version is not an exported symbol. `const mlwhAPIVersion = "1.6.0"` (mlwh/openapi.go) is unexported; the only public access is walking the untyped map returned by `OpenAPIDocument()` at `info.version`. Export the version as a public compile-time symbol (e.g. `const APIVersion`) and have `mlwhAPIVersion` / the OpenAPI `info.version` derive from that single constant so the public symbol and served document cannot drift. Reporting it must not require contacting a live server.
    - mlwh/openapi.go: added exported, doc-commented `const APIVersion = "1.6.0"` (compiled-in); `mlwhAPIVersion` and the OpenAPI `info.version` now derive from that single constant, so the public symbol and served document cannot drift.
    - mlwh/openapi_test.go: added TDD GoConvey white-box test `TestOpenAPIPublicVersionConstantNoDrift` asserting `APIVersion` is non-blank and equals the served `info.version`. Existing `TestOpenAPIDocumentIdentityC2` (info.version == mlwhAPIVersion) left intact.
    - Additive only: value unchanged at "1.6.0"; Registry, served API, and api-reference.md golden tests untouched. Reviewer PASS; `go test ./mlwh/...` and `make lint-go` green.
- [x] BUG 2 — RemoteClient has no exported generic dispatcher. `(rc *RemoteClient) do(ctx, method, pathParams, query)` (mlwh/remote.go) is the ideal Registry-keyed dispatcher (looks the Endpoint up by Method name, builds the URL, decodes into the typed NewResult()), but is unexported, so a consumer selecting an endpoint dynamically by `mlwh.Registry` Method name (an MCP "call any endpoint" escape-hatch tool) cannot dispatch through the typed client without reflection or a ~40-arm switch. Add exported `(rc *RemoteClient) Call(ctx context.Context, method string, pathParams []string, query url.Values) (any, error)` delegating to `do`, returning the decoded typed result as `any` and surfacing the existing unknown-method and path-param-arity errors.
    - mlwh/remote.go: added exported thin wrapper `Call(ctx, method, pathParams, query) (any, error)` next to `do`, delegating to it. `do`, the Registry, and the typed methods are unchanged. Returns the decoded typed result (a pointer) as `any` and surfaces `do`'s unknown-method and path-param-arity errors.
    - mlwh/remote_test.go: added 4 TDD hermetic httptest GoConvey tests — equivalence with typed ResolveSample, pagination passthrough (limit/offset via url.Values), unknown-method error, path-param arity-mismatch error.
    - Additive (+18/-0 remote.go, +91/-0 test). `Call` is a method on `*RemoteClient` only, not on the `Queryer` interface, so the reflection-pinned parity / OpenAPI no-drift golden tests are unaffected. Reviewer PASS; `go test ./mlwh/...` and `make lint-go` green.
