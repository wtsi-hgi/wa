- [x] Opening a results page gives node error "Hydration failed because the server rendered text didn't match the client. As a result this tree will be regenerated on the client. This can happen if a SSR-ed Client Component used: […] +aria-label="enrichment unavailable" [..] - aria-label="loading enrichment""
    - Root cause: `ResultMetadataEnrichment` read `SeqmetaCacheContext` directly during render; `not_found` cookie entries hydrated on client made the first client render diverge from SSR.
    - Fix: gate cache reads behind a hydration-safe mounted flag via `useSyncExternalStore` (server snapshot returns `false`), using a module-scoped `EMPTY_CACHE` until mounted; effect also gated.
    - Files: `frontend/components/result-metadata-enrichment.tsx`, regression tests added in `frontend/tests/seqmeta-badge.test.ts`.
- [x] File preview truncation is broken following recent commits and attempts in prior 4 bugfixes md files to improve this. All 81979 rows of a large tsv are still shown in the file browser, ignoring the preview height and limits on how much data to send to the browser. Clicking to enlarge the preview correctly shows pagination, but the file browser preview should not have been given all the rows to render. Only enough rows to fill the preview height (or the maximum possible preview height, with height slider adjustements then not having to trigger fresh data requests). Learn from the mistakes of prior attempts at fixing this. How can this be so difficult? The backend just shouldn't send more than approx 20 lines of data when sending file browser previews, obey its max size limits and truncate when sending the data for enlarged previews, and send whole files for the download file button.
    - Root cause: a single backend preview path served all surfaces; prior fixes patched only the frontend and could be overridden by a client-supplied `line_limit`.
    - Fix: introduce `mode` query param with three server-enforced modes — `inline` (hard cap `MaxInlinePreviewLines=20`, ignores larger client `line_limit`), `enlarged` (byte-cap only, paginated client-side), `download` (full file, attachment).
    - Frontend inline preview requests `mode=inline`; height-slider changes no longer trigger refetches (excluded from effect deps); enlarge fetches `mode=enlarged` once on open; download untouched.
    - Files: `results/server_file.go`, `results/server_file_test.go`, `frontend/app/api/file/route.ts`, `frontend/components/file-preview.tsx`, `frontend/components/result-detail-files.tsx`, `frontend/tests/result-detail-files.test.ts`, `frontend/e2e/results.spec.ts`, `frontend/e2e/file-preview-truncation.spec.ts`.
