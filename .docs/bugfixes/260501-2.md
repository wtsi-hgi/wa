# Bugfix 260501-2: Compact preview height selector placement

**Status:** Fixed  
**Branch:** results-rest  
**Date:** 2026-05-01

## Problem

In the file browser, the preview height selector was taking up considerable vertical space in the header section and was separated from related controls. It should be minimized in size and placed next to the "1 preview per row" toggle that appears on folders containing files.

## Solution

### UI Changes

1. **Redesigned PreviewHeightControl component** to be compact and inline:
   - Changed from block layout (`block rounded-[1.25rem] px-4 py-3`) to inline-flex (`inline-flex items-center gap-2 rounded-full px-3 py-2`)
   - Reduced slider dimensions from full-width with top margin to compact inline (`h-1 w-24`)
   - Added Eye icon to match the visual style of the toggle control
   - Redesigned value display to be compact with tabular numbers
   - Maintained all interaction functionality (drag, keyboard, blur commits)

2. **Relocated control placement**:
   - Removed from file browser header section
   - Integrated into `renderPreviewControls` function
   - Now appears next to "1 preview per row" toggle when `placement === "folder"`
   - Shares the same visual container with pagination controls

3. **Simplified header**:
   - Removed the separate control section that held the preview height slider
   - Header now only contains the "File Browser" label and icon

### Test Changes

1. **Updated existing unit test** (`surfaces preview height without putting paging controls in the browser header`):
   - Added `renderGridPreview` prop to satisfy control visibility conditions
   - Added explicit verification that preview height control is NOT in header
   - Added verification that control IS in folder controls section
   - Verified both "Preview height" and "1 preview per row" appear together

2. **Added e2e regression test** (`places the compact preview height selector next to the 1 preview per row toggle in folder controls`):
   - Verifies preview height slider exists in folder controls container
   - Checks controls are positioned on the same horizontal line
   - Validates slider height is compact (< 80px)
   - Confirms overall controls container height is reasonable (< 120px)
